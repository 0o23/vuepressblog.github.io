# 建站记录

# 一、概述

本博客是用 Vuepress 框架建立的，这个框架很容易上手，操作起来也比较方便。唯一一个不太熟练的地方就是博客的部署，这个也是本人缺乏部署经验导致的，Vuepress 还是很好用的。为了更好地学习和复习 Vuepress，特此将学习过程记录下来。注：此博客主要是给本人复习所用，并不太适合作为萌新入门文章。

# 二、必备工具

node (opens new window)

安装完 node 后，打开命令窗口，输入这两个命令，查看是否安装完成：

node --version

npm --version

# 三、正式开始

# 1. 使用 node 自带的 npm 下载 Vuepress

打开命令窗口，输入：

npm install -g vuepress

# 查看是否安装成功
vuepress --version

# 3. 目录文件简单介绍

  1. package.json——项目的描述文件，是用 npm init 命令自动生成的

  1. docs/README.md——项目主页显示的内容。注：在 Vuepress 中，页面显示的内容都是依赖 .md 文件的，因此每个文件夹(可以把文件夹看做是页面)都必须有一个 README.md 文件，负责展示该页面的内容。

# 4. 项目文件详细介绍

详细介绍主要说明的是这些文件内可写的属性，其实也不会介绍很详细，更多内容请查看官方文档 (opens new window)还有这个 (opens new window)

# 1. docs/README.md

  • 这个文档的内容就不多说了，对照着主页的显示就可以很容易得看出来这些属性各自的效果。

# 2. docs/.vuepress/config.js


1). base: String (/)

  • base 是部署站点的基础路径，如果你想让你的网站部署到一个子路径下，你将需要设置它。如果你想将你的网站部署到 https://foo.github.io/bar/，那么 base 应该被设置成 "/bar/"

2). title: String (undefined)

  • 网站的标题，它将会被用作所有页面标题的前缀，同时，默认主题下，它将显示在导航栏（navbar）上。

3). description: String (undefined)

  • 网站的描述，它将会以 <meta> 标签渲染到当前页面的 HTML 中。

4). host: String ('0.0.0.0')

  • 指定用于 dev server 的主机名。

5). port: String (8080)

  • 指定 dev server 的端口。

6). head: Array ([])

  • 额外的需要被注入到当前页面的 HTML 中的 <head> 标签

例如：
head: [ ['link', {rel: 'icon', href: '/img/logo.png'}], ],
相当于在最终的 index.html 页面中添加了： <link rel='icon' href='/img/logo.png'>



7). markdown.lineNumbers: boolean (undefined)

  • 是否在每个代码块的左侧显示行号。

8). themeConfig: Object ({})

  • 为当前的主题提供一些配置，这些选项依赖于你正在使用的主题。

9). themeConfig.nav: Array ([])

  • 增加一些导航栏链接。

10). themeConfig.sidebar: Array ([])

  • 增加一些侧边栏。
