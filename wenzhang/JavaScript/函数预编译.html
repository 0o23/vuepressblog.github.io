# 函数预编译问题：

问题如下：

function fn(a, c) {
  var a;
  var b;
  console.log(a);
  a = 123;
  console.log(a);
  console.log(c);
  function a() { }
  if (false) {
    var d = 678;
  }
  console.log(d);
  console.log(b);
  b = function () { }
  console.log(b);
  function c() { }
  console.log(c);
}

fn(1, 2);

讲解：

  1. 预编译——即作用域的创建阶段（本题专指函数的作用域创建阶段）

  2. 函数预编译的时候做了哪些事情呢？

step1. 函数作用域创建阶段会生成一个 JS 的变量对象，它也被称为 AO 对象。AO 对象是供 JS 引擎访问的，我们是访问不到的；

step2. 找函数的形参和函数内的变量的声明，并将其作为 AO 对象的属性名，对应的属性值是 undefined；

step3. 形参和实参相统一；

step4. 找函数声明，并且，函数声明的名称和变量声明一致的话，函数声明会覆盖变量声明。



根据上面的讲解，解决问题就水到渠成了：

step1. 生成 AO 对象：

AO: { }
1

step2. 形参和变量声明是 AO 对象的属性，初始属性值为 undefined

AO: {
  a: undefined
  c: undefined
  b: undefined
  d: undefined
}

AO: {
  a: undefined ---> 1 ---> function a() { }
  c: undefined ---> 2 ---> function c() { }
  // 注：b = function () { } 不是函数声明，是变量声明
  b: undefined

对照 AO 对象，得到本题函数每行的输出结果为：

function fn(a, c) {
  var a;
  var b;
  console.log(a); // function a() { }
  a = 123;
  console.log(a); // 123
  console.log(c); // function c() { }
  function a() { }
  if (false) {
    var d = 678;
  }
  console.log(d); // undefined
  console.log(b); // undefined
  b = function () { }
  console.log(b); // function() { }
  function c() { }
  console.log(c); // function c() { }
}
